# Framework di Valutazione

La valutazione nel nostro dipartimento è **autentica, continua e trasparente**.
Non si valuta la memoria: si valutano le **competenze in azione**.

## Principi della nostra valutazione

1. **Trasparenza:** I criteri di valutazione sono condivisi PRIMA dell'inizio del progetto
2. **Continuità:** Si valuta durante tutto il processo, non solo alla fine
3. **Autenticità:** Il prodotto reale è la prova di competenza
4. **Molteplicità:** Prodotto + processo + auto-valutazione + peer assessment
5. **Feedback formativo:** Il voto è meno importante del feedback su come migliorare

---

## Struttura della valutazione per progetto

```
Peso tipico per un progetto semestrale:

30% → Prodotto finale (funziona? È di qualità?)
25% → Qualità tecnica (codice, architettura, documentazione)
20% → Presentazione / demo al committente
15% → Processo (sprint, commit, retrospettiva)
10% → Uso critico e documentato dell'AI
```

---

## Raccordo con il registro scolastico

I voti sul registro vengono assegnati in questi momenti:
- **Fine sprint 1** → voto di processo (partecipazione, organizzazione)
- **Demo intermedia** → voto di prodotto parziale
- **Demo finale** → voto di prodotto completo
- **Presentazione orale** → voto di comunicazione
- **Auto-valutazione/peer** → contribuisce alla media finale
