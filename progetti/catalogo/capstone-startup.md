# Progetto Capstone: Startup Digitale

| Campo | Valore |
|-------|--------|
| **Titolo** | Capstone – Dal problema al prodotto: la nostra startup digitale |
| **Anno di corso** | 5° |
| **Durata** | Settembre – Maggio (8 mesi) |
| **Team size** | 3–4 studenti |
| **Stato** | `attivo` |
| **Referente** | Docenti A042 (TPSIT + GPOI) |

---

## Il Problema

Il 5° anno è il momento in cui gli studenti **dimostrano di saper fare tutto**:
progettare, costruire, testare, documentare, presentare e gestire un prodotto digitale completo.

Il problema: **costruire un prodotto digitale reale che usa AI, è deployato in produzione,
rispetta il GDPR e l'EU AI Act, e potrebbe essere effettivamente lanciato come startup.**

---

## Committente / Stakeholder

- **Azienda / ente del territorio** (obbligatorio – accordo entro settembre)
- **Commissione di Esame di Stato** (giugno)
- Potenziali investitori / mentori (demo day facoltativo)

---

## Output attesi

- [x] Prodotto funzionante e **deployato in produzione** (URL pubblico)
- [x] Codebase su GitHub con CI/CD funzionante
- [x] Documentazione tecnica completa (README, architettura, API docs, ADR)
- [x] Business plan sintetico (problem/solution fit, target, revenue model)
- [x] Pitch deck (10 slide, stile startup)
- [x] Documento di compliance GDPR + EU AI Act (se rilevante)
- [x] Relazione finale per l'Esame di Stato

---

## Macro-fasi

| Fase | Descrizione | Durata |
|------|-------------|--------|
| 1. Kick-off | Scelta progetto, accordo committente, team formation | 2 sett. |
| 2. Discovery | Analisi requisiti, architettura, business model | 2 sett. |
| 3. Sprint 1–6 | Sviluppo iterativo con sprint bisettimanali | 12 sett. |
| 4. Alpha release | Prodotto funzionante, test interni | 2 sett. |
| 5. Beta release | Test con utenti reali, feedback, fix | 2 sett. |
| 6. Production | Deploy, monitoring, documentazione finale | 2 sett. |
| 7. Demo Day | Presentazione pubblica a committenti e scuola | 1 sett. |
| 8. Esame di Stato | Presentazione al colloquio | Giugno |

---

## Ruoli nel team (obbligatori)

- **Tech Lead:** architettura, code review, decisioni tecniche
- **Full-stack Developer:** sviluppo core del prodotto
- **AI/Data Engineer:** componente AI del prodotto
- **PM + Business:** project management, business plan, relazioni committente

## Strumenti e tecnologie

- **Stack:** libero (scelto dal team in base al problema)
- **AI:** obbligatoria in modo significativo nel prodotto
- **DevOps:** GitHub Actions (CI/CD), Docker, cloud provider (AWS/Azure/GCP for Students)
- **PM:** GitHub Projects o Notion (Scrum board)
- **Monitoring:** Sentry, Grafana o simili

---

## Raccordo con l'Esame di Stato

Il progetto capstone è il **centro del colloquio**:
1. Il candidato presenta il progetto (10 min)
2. La commissione approfondisce aspetti tecnici, etici, organizzativi
3. Si collegano tutte le discipline del quinquennio al progetto

Ogni studente prepara una **relazione individuale** che descrive il suo contributo specifico
al progetto di team.
