# Progetto: AI Tool Reale

| Campo | Valore |
|-------|--------|
| **Titolo** | AI Tool Reale – Costruire uno strumento AI per un problema concreto |
| **Anno di corso** | 4° |
| **Durata** | Ottobre – Maggio (7 mesi) |
| **Team size** | 3–4 studenti |
| **Stato** | `attivo` |
| **Referente** | Docente A042 |

---

## Il Problema

L'AI è ovunque, ma pochi la capiscono davvero.
Il progetto chiede agli studenti: **quale problema reale possiamo risolvere costruendo
uno strumento che usa l'AI in modo significativo e responsabile?**

Il problema deve venire dal territorio: un'azienda, un ente, la scuola stessa.

---

## Idee di progetto (esempi da anni precedenti)

- **MensaAI:** classificatore di immagini per identificare i piatti della mensa e stimare calorie
- **OrientaBot:** chatbot per l'orientamento scolastico rivolto agli studenti delle medie
- **SentimentSchool:** analisi del sentiment nelle comunicazioni scuola-famiglia
- **AccessibilityReader:** riconoscimento vocale per studenti con DSA
- **CropMonitor:** AI per il monitoraggio di colture locali (in collaborazione con agrario)

---

## Committente / Stakeholder

- **Organizzazione reale** (da trovare nel territorio entro il primo mese)
- Il committente partecipa alle milestone e alla demo finale

---

## Output attesi

- [x] Applicazione funzionante con AI integrata (non solo API call, ma pipeline completa)
- [x] Dataset documentato (fonte, licenza, analisi bias)
- [x] Report prestazioni del modello (metriche, confusion matrix, limiti)
- [x] Documento etico: rischi, bias identificati, mitigazioni
- [x] Demo pubblica con committente presente
- [x] Codice open source su GitHub con documentazione

---

## Macro-fasi

| Fase | Descrizione | Durata |
|------|-------------|--------|
| 1. Problem framing | Definizione precisa del problema con committente | 2 sett. |
| 2. Data exploration | Raccolta, pulizia, analisi del dataset | 3 sett. |
| 3. Baseline model | Primo modello semplice, metriche base | 3 sett. |
| 4. Improvement | Iterazione sul modello, feature engineering | 3 sett. |
| 5. Integration | Integrazione in app usabile (web/mobile) | 3 sett. |
| 6. Ethical review | Analisi bias, rischi, documento etico | 1 sett. |
| 7. Demo & Docs | Demo pubblica, documentazione finale | 2 sett. |

## Strumenti e tecnologie

- **ML:** Python, scikit-learn, TensorFlow/Keras o PyTorch (base), Pandas, NumPy
- **AI generativa:** OpenAI API, HuggingFace, Ollama (locale)
- **Sviluppo:** VS Code, Jupyter, Google Colab
- **Deployment:** Streamlit, Gradio, o Flask + React
- **Cloud:** Google Colab Pro, Azure for Students, Hugging Face Spaces
