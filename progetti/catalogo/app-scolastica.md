# Progetto: App Scolastica

| Campo | Valore |
|-------|--------|
| **Titolo** | App Scolastica – Risolviamo un problema reale della nostra scuola |
| **Anno di corso** | 2° |
| **Durata** | Ottobre – Maggio (7 mesi) |
| **Team size** | 3–4 studenti |
| **Stato** | `attivo` |
| **Referente** | Docente A041 |

---

## Il Problema

La scuola ha tanti piccoli problemi organizzativi irrisolti.
Il progetto chiede agli studenti: **quale problema della nostra scuola possiamo risolvere con un'app?**

Esempi di problemi reali identificati dalle classi precedenti:
- Prenotazione aula computer
- Gestione attività extrascolastiche
- Notifiche eventi ai genitori
- Monitoraggio prestiti libri/strumenti
- Registro presenza attività sportiva

---

## Committente / Stakeholder

- **Personale scolastico** (ATA, docenti, dirigenza) – committenti reali
- Studenti della scuola – utenti finali
- Genitori (in alcuni scenari)

---

## Output attesi

- [x] Applicazione funzionante (Python + database)
- [x] Interfaccia utente (grafica o web, a scelta del team)
- [x] README tecnico e documentazione
- [x] Presentazione al committente (personale scolastico)
- [x] Retrospettiva di team documentata
- [x] Repository GitHub del team con storia dei commit

---

## Macro-fasi (Scrum semplificato)

| Fase | Descrizione | Durata |
|------|-------------|--------|
| Sprint 0 | Analisi dei bisogni, interviste al personale, scelta del problema | 2 sett. |
| Sprint 1 | Architettura, DB design, prototipo interfaccia | 2 sett. |
| Sprint 2 | Core features (CRUD base) | 2 sett. |
| Sprint 3 | Features avanzate, gestione errori | 2 sett. |
| Sprint 4 | Testing, bug fixing, ottimizzazione | 2 sett. |
| Sprint 5 | Deploy, documentazione, demo al committente | 2 sett. |
| Retrospettiva | Analisi del processo, lezioni apprese | 1 sett. |

**Daily stand-up:** 10 minuti a inizio lezione (cosa ho fatto, cosa farò, blocchi)

## Strumenti e tecnologie

- **Sviluppo:** Python, SQLite (poi PostgreSQL), Tkinter o Flask (interfaccia)
- **AI:** GitHub Copilot (codice), ChatGPT (debug, spiegazioni, test cases)
- **Collaborazione:** GitHub Projects (Kanban), Git branching workflow
- **Deployment:** Locale o su Raspberry Pi della scuola
